# Combined bundle format

## Overview
At app launch, OnlookMobileClient composes two JS bundles into one file that Hermes evaluates. This doc specifies the byte-level layout.

## Components
1. **`onlook-runtime.js`** — built from `packages/mobile-preview/runtime/entry.js` via `packages/mobile-preview/server/build-runtime.ts`. Ships React + reconciler + B13 shell. Output size: ~257KB (263,069 bytes in the current build; 231 lines). Installed into the `.app` Resources bundle at `Resources/onlook-runtime.js` via `apps/mobile-client/scripts/bundle-runtime.ts`.

2. **`main.jsbundle`** — the Expo/Metro-produced user bundle. Generated by `expo export:embed` or Metro dev server. Size varies (~1.3MB for a minimal app).

## Composition (at launch)
`ReactNativeDelegate.bundleURL()` in `AppDelegate.swift` is the single seam where composition happens (bridgeless / new-arch mode: `RCTHost` consults `bundleURL()` and never `loadSource:` / `loadBundleAtURL:`):
1. Reads the baked user bundle: `Bundle.main.url(forResource: "main", withExtension: "jsbundle")` via `Data(contentsOf:options: .mappedIfSafe)`. If missing, falls through to Metro (DEBUG only) with runtime injection skipped.
2. Calls `HermesBootstrap.prepend(into: userData)`, which in turn reads `Bundle.main.url(forResource: "onlook-runtime", withExtension: "js")` via `Data(contentsOf:options: .mappedIfSafe)`.
3. `HermesBootstrap.prepend` allocates `Data(capacity: runtime.count + userData.count + 1)` and appends in order:
   - Byte `0..N-1`: `onlook-runtime.js` content
   - Byte `N`: separator — a single byte `0x0A` (`'\n'`)
   - Byte `N+1..M`: `main.jsbundle` content
4. Writes the result to `NSTemporaryDirectory() + "onlook-combined.jsbundle"` with `.atomic` options.
5. Returns the combined file URL to Expo/RN, which passes it to `RCTHost`.

On success, `HermesBootstrap.prepend` emits `[onlook-runtime] hermes ready` via `OnlookLogger.notice` (Apple unified log `.default` level so `log show` persists it without `--info`). On runtime load failure it logs an error and returns `userData` unchanged. If bundle composition throws, `AppDelegate` falls back to returning the baked user bundle URL alone.

## Evaluation order
Hermes evaluates the combined bundle as a single script:
1. Runtime IIFE runs first — sets globals (`global.OnlookRuntime`, `global.React`), installs host objects, populates the Metro `__d` / `__r` module registry and JSI polyfills the user bundle expects.
2. Main bundle runs second — uses globals via normal `require`.
3. Main bundle triggers `runApplication(bundleSource)` via MC2.7.

## Line-number interpretation
Crash traces from Hermes reference combined-bundle line numbers. With the current shipped runtime (231 lines):
- Lines `1..231` = runtime (line numbers match `packages/mobile-preview/runtime/bundle.js` at its shipped 231-line form; the dev-built source currently totals 233 lines — verify against the exact `onlook-runtime.js` baked into the `.app`).
- Line `232` = separator (`\n`).
- Lines `233+` = main bundle. Line `N` of combined = line `N - 232` of `main.jsbundle`.

When debugging, translate the combined-bundle line to source via:
- If line < 232: check the baked `Resources/onlook-runtime.js` (mirrors `packages/mobile-preview/runtime/bundle.js`).
- If line >= 233: subtract 232 and check `main.jsbundle` (or source-map if available).

Note: the 231-line count is a property of the current build output, not a stable contract. Re-derive with `wc -l Resources/onlook-runtime.js` when reading a stack trace.

## Debugging the combined bundle
The combined file is written to `NSTemporaryDirectory()` on every launch. To inspect:
1. On device: hard to access (sandbox). Use `xcrun devicectl device info files` for iOS 17+.
2. On simulator: `~/Library/Developer/CoreSimulator/Devices/<UUID>/data/tmp/onlook-combined.jsbundle`.
3. On Mac: the runtime source is always at `packages/mobile-preview/runtime/bundle.js`; combine manually with: `cat bundle.js main.jsbundle > test-combined.js` (note this uses `cat` which joins without adding a separator — the real build inserts a `\n` byte between them; for an exact reproduction use `{ cat bundle.js; printf '\n'; cat main.jsbundle; } > test-combined.js`).

`AppDelegate.bundleURL()` also logs `[onlook-runtime] composed combined bundle (<bytes> bytes) at <path>` via `NSLog` on every successful compose, so the exact path + size is discoverable in `log show`.

## Size constraints
- Runtime ~257KB (constant per session; 263,069 bytes at time of writing).
- User bundle varies; MCI.3's 20KB gate applies to Onlook-target bundles specifically.
- Combined ~1.57MB for current scaffold.
